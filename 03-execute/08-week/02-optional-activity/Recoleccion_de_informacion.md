# 🌱 AgroWeb Huila  
## Instrumentos de Recolección de Información

---

## 📌 Descripción del Proyecto

AgroWeb Huila es un sistema web enfocado en la gestión y monitoreo de cultivos de café.  
Su objetivo principal es ayudar a los agricultores a llevar un mejor control de sus actividades diarias, como el riego, la fertilización y el seguimiento del crecimiento de las plantas.

Antes de desarrollar la plataforma, se realizó un proceso de recolección de información para entender cómo trabajan actualmente los agricultores y cuáles son sus principales necesidades.

Este proceso permitió identificar:

- Cómo registran la información actualmente  
- Qué dificultades tienen  
- Qué datos necesitan guardar  
- Qué tipo de herramienta tecnológica sería más útil  

---

## 🎯 Objetivo de la Recolección de Información

El objetivo fue obtener información real del entorno agrícola para diseñar un sistema que realmente solucione problemas y no sea solo una idea teórica.

---

## 🧩 Instrumentos Aplicados

### 🗣️ 1. Entrevista

La entrevista se realizó directamente con agricultores de la región, permitiendo conocer su experiencia y forma de trabajo.

#### 📌 Preguntas realizadas

- ¿Cómo registra el riego de sus cultivos?  
- ¿Cada cuánto aplica fertilizantes?  
- ¿Dónde guarda la información del cultivo?  
- ¿Qué problemas ha tenido al llevar el control manual?  
- ¿Le gustaría usar una herramienta digital?  

#### 🎯 Resultados

Se identificó que:

- Usan cuadernos o hojas sueltas  
- No tienen respaldo de información  
- A veces olvidan datos importantes  
- Se pierde el historial del cultivo  

#### 💡 Aplicación en el sistema

A partir de esto, AgroWeb incluye:

- Registro digital de actividades  
- Almacenamiento en base de datos  
- Historial organizado por fechas  
- Acceso rápido a la información  

---

### 📊 2. Encuesta

Se aplicaron encuestas a varios agricultores para obtener datos más generales sobre el tipo de usuario.

#### 📌 Preguntas realizadas

- ¿Cuántas veces riega por semana?  
- ¿Cuántas plantas tiene?  
- ¿Utiliza celular o computador?  
- ¿Tiene acceso a internet?  
- ¿Ha usado alguna aplicación web antes?  

#### 🎯 Resultados

- La mayoría usa teléfonos móviles  
- Algunos tienen poco conocimiento tecnológico  
- Prefieren herramientas simples  
- El acceso a computador es limitado  

#### 💡 Aplicación en el sistema

Con base en esto, AgroWeb será:

- Responsive (adaptado a celular)  
- Fácil de usar (interfaz simple)  
- Con botones claros y visibles  
- Con procesos cortos y rápidos  

---

### 👀 3. Observación

Se realizó observación directa del trabajo en los cultivos para entender mejor el proceso real.

#### 📌 Actividades observadas

- Revisión del estado de las plantas  
- Aplicación de riego  
- Uso de fertilizantes  
- Registro manual de datos  
- Seguimiento del crecimiento  

#### 🎯 Resultados

Se identificó el flujo real de trabajo:

1. Revisar cultivo  
2. Regar  
3. Fertilizar  
4. Registrar información  

#### 💡 Aplicación en el sistema

El sistema se diseñará siguiendo ese mismo flujo para que sea más intuitivo.

---

## ⚙️ Requerimientos Identificados

### 🔹 Funcionales

- Registrar cultivos  
- Registrar riego  
- Registrar fertilización  
- Consultar historial  
- Ver seguimiento del crecimiento  
- Editar y eliminar registros  

### 🔹 No Funcionales

- Fácil de usar  
- Accesible desde celular  
- Rápido  
- Seguro (datos protegidos)  
- Interfaz clara  

---

## 🧠 Problemas Detectados

Durante la recolección se encontraron varios problemas:

- Pérdida de información  
- Desorganización  
- Falta de seguimiento  
- Dependencia de la memoria  
- Dificultad para tomar decisiones  

---

## 💡 Solución Propuesta

AgroWeb Huila busca solucionar estos problemas mediante:

- Digitalización de datos  
- Organización automática de información  
- Historial de actividades  
- Acceso desde cualquier lugar  
- Mejora en la toma de decisiones  

---

La aplicación de entrevistas, encuestas y observación permitió entender la realidad del trabajo agrícola y diseñar un sistema basado en necesidades reales.
